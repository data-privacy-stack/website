Contributions of all sizes are welcome — code, documentation, bug reports,
feature ideas, and reviews.

## Ways to contribute

- **Report a bug or request a feature.** Open an issue in the relevant project
  repository under the [Data Privacy Stack organization](https://github.com/data-privacy-stack).
- **Improve the docs.** Each project's documentation is maintained in its own
  repository; this portal site lives in
  [`data-privacy-stack/website`](https://github.com/data-privacy-stack/website).
- **Submit a pull request.** Pick up an open issue, or propose a change. Small,
  focused PRs are easiest to review.
- **Join the discussion.** Share use cases, ask questions, and help shape the
  roadmap.

## Working on Presidio

Presidio is developed at
[`data-privacy-stack/presidio`](https://github.com/data-privacy-stack/presidio).
See the repository's `CONTRIBUTING.md` for development setup, testing, and the
pull-request process.

## Proposing a new project

The stack is intended to grow. If you maintain — or want to start — a
privacy-focused open-source project that fits, open a discussion or issue so the
community can weigh in. See [governance](/community/governance/) for how
decisions are made.

## Code of conduct

All participation is governed by the project's code of conduct. Be respectful,
assume good intent, and help keep the community welcoming.
